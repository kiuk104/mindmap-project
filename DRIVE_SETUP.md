# 구글 드라이브 연동 설정 가이드

마인드맵을 구글 드라이브에 직접 저장/불러오기하려면 한 번 OAuth 클라이언트를 설정해야 합니다.
**5분 정도면 끝납니다.**

## 1단계 — Google Cloud 프로젝트 생성

1. https://console.cloud.google.com 접속
2. 상단 프로젝트 드롭다운 → **새 프로젝트**
3. 프로젝트 이름: 예) `mindmap-project` → **만들기**

## 2단계 — Google Drive API 활성화

1. 좌측 메뉴 → **API 및 서비스** → **라이브러리**
2. `Google Drive API` 검색 → 클릭 → **사용** 버튼

## 3단계 — OAuth 동의 화면 구성

1. 좌측 메뉴 → **API 및 서비스** → **OAuth 동의 화면**
2. **외부(External)** 선택 → 만들기
3. 앱 정보 입력:
   - **앱 이름**: 마인드맵 (자유)
   - **사용자 지원 이메일**: 본인 이메일
   - **개발자 연락처**: 본인 이메일
4. **저장 후 계속** → 범위 단계는 그대로 → 테스트 사용자에 본인 Gmail 추가 → 저장

> 검증되지 않은 앱이라는 경고가 나오지만, 본인 계정으로만 쓸 거라서 무시해도 됩니다.

## 4단계 — OAuth 2.0 클라이언트 ID 생성

1. 좌측 메뉴 → **API 및 서비스** → **사용자 인증 정보**
2. **사용자 인증 정보 만들기** → **OAuth 클라이언트 ID**
3. **애플리케이션 유형**: `웹 애플리케이션`
4. **이름**: 예) `mindmap-web`
5. **승인된 자바스크립트 원본** — 아래 두 URL을 모두 추가:
   ```
   http://localhost:5173
   https://kiuk104.github.io
   ```
   *(다른 호스트도 사용한다면 함께 추가)*
6. **승인된 리디렉션 URI**는 비워둬도 됩니다 (이 앱은 팝업 방식 사용)
7. **만들기** → 표시되는 **클라이언트 ID** 복사

## 5단계 — 코드에 붙여넣기

`src/config.js` 파일을 열어 빈 문자열에 클라이언트 ID를 넣습니다:

```js
export const GOOGLE_CLIENT_ID = '123456789-abcdefg.apps.googleusercontent.com';
```

저장 후 다음을 실행:

```bash
git add src/config.js
git commit -m "chore: configure Google Drive OAuth client"
git push
```

GitHub Actions가 자동으로 빌드 → 배포합니다.

> **참고:** OAuth 클라이언트 ID는 공개되어도 안전합니다.
> 보안은 4-5단계에서 등록한 "승인된 자바스크립트 원본"으로 통제됩니다.
> 다른 도메인에서는 이 ID가 동작하지 않습니다.

## 6단계 — 사용

1. https://kiuk104.github.io/mindmap-project/ 접속 (또는 로컬 http://localhost:5173 )
2. 툴바의 **☁️ Drive 연결** 클릭
3. 구글 로그인 + 권한 동의 (마인드맵 앱이 만든 파일만 접근)
4. 연결되면 버튼이 `✅ 이메일` 로 바뀜
5. **💾 저장** → 위치를 "구글 드라이브"로 선택
6. **☁️ 드라이브 불러오기** 로 저장된 마인드맵 목록 확인

## 권한 범위 (Scope)

이 앱은 `drive.file` scope만 요청합니다:

- ✅ 이 앱이 만든 파일을 읽고 쓰기
- ❌ 사용자의 다른 Drive 파일에는 절대 접근 불가
- ❌ 폴더 전체 검색 불가

OAuth 동의 화면에서도 "마인드맵 앱으로 만든 파일을 보고 관리할 수 있습니다"로만 표시됩니다.

## 문제 해결

| 증상 | 원인 / 해결 |
|------|-----|
| 버튼이 "☁️ Drive 설정 필요"로 표시됨 | `src/config.js`의 `GOOGLE_CLIENT_ID`가 비어 있음 |
| `403 redirect_uri_mismatch` | 승인된 자바스크립트 원본에 현재 URL이 없음 |
| `idpiframe_initialization_failed` | 쿠키 차단 해제 또는 다른 브라우저 시도 |
| "검증되지 않은 앱" 경고 | OAuth 동의 화면 테스트 사용자에 본인 추가 안 됨 |
| 1시간 후 자동 로그아웃 | 정상 동작 — 토큰 만료. 다시 "Drive 연결" 클릭 |
